Overview

SynthList is a Roblox script designed for secure key-based authentication and Lua script execution. It includes features for HWID (Hardware ID) validation, local key storage, customizable notifications, and enhanced debugging.

Key Features

  • Key-Based Authentication: Requires a valid key to execute the script.
  • HWID Validation: Checks and optionally updates the HWID to prevent unauthorized access.
  • Local Key Storage: Saves keys locally to avoid re-entry.
  • Customizable Notifications: Allows customization or disabling of notifications.
  • Debug Mode: Provides additional logging for debugging purposes.
  • Flexible Settings: Features such as HWID updates and notifications can be toggled.

Configuration

Required Global Variables

  • _G.USERNAME: Username for API access.
  • _G.PASSWORD: Password for secure API access.
  • _G.KEY: Key entered by the user.
  • _G.SCRIPT: Lua script to be executed upon successful authentication.

Optional Global Variables

  • _G.FILENAME: Custom filename for saving the key locally (default is "Saved__Key.json").
  • _G.SAVEKEY: Set to true to enable saving the key locally (default is true).
  • _G.UPDATEHWID: Set to false to disable HWID updates (default is true).
  • _G.NOTIFY_ON_SUCCESS: Set to false to disable success notifications (default is true).
  • _G.NOTIFY_ON_ERROR: Set to false to disable error notifications (default is true).
  • _G.CUSTOM_NOTIFY: Custom function to handle notifications.
  • _G.DEBUG_MODE: Set to true to enable debug logging (default is false).

How It Works

  • Fetching Keys and HWIDs: The script requests keys and HWIDs from the server and parses the response to validate the user.
  • HWID Validation and Update: Checks if the HWID associated with the key is valid and updates it if necessary, based on the _G.UPDATEHWID setting.
  • Key Storage: Saves, loads, and deletes the key locally based on the settings in _G.SAVEKEY.
  • Notifications: Displays notifications based on user preferences. Custom notification functions can be provided through _G.CUSTOM_NOTIFY.
  • Script Execution: Executes the Lua script specified in _G.SCRIPT after validating the key and HWID.
  • Debug Logging: If enabled by _G.DEBUG_MODE, logs additional debug information for troubleshooting.

Example Usage

  • _G.USERNAME = "yourUsername"
_G.PASSWORD = "yourPassword"
_G.KEY = "yourKey"
_G.SCRIPT = "print('Hello, world!')"
_G.FILENAME = "MyCustomKey.json"
_G.UPDATEHWID = true
_G.NOTIFY_ON_SUCCESS = true
_G.NOTIFY_ON_ERROR = true
_G.DEBUG_MODE = true  -- Enable debug logging
-- Custom notification function
_G.CUSTOM_NOTIFY = function(title, text)
    print("Custom Notification:", title, text)
end

    • Troubleshooting

    Issue: Script fails to authenticate.
    Solution: Verify that the username, password, and key are correctly entered and that the server is reachable.
    Issue: HWID update not working.
    Solution: Ensure _G.UPDATEHWID is set to true and that the server accepts HWID updates.
    Issue: Notifications not appearing.
    Solution: Check if _G.NOTIFY_ON_SUCCESS and _G.NOTIFY_ON_ERROR are enabled and ensure the CUSTOM_NOTIFY function is properly defined.
    Issue: Debug information not showing.
    Solution: Ensure _G.DEBUG_MODE is set to true for additional logging.

    Summary

    • SynthList provides a secure and customizable solution for key-based authentication in Roblox. By adjusting the global variables, users can tailor the script to their needs, ensuring a flexible and efficient experience.
